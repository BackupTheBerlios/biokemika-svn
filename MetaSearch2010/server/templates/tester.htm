Trigger-Test-Suite

Auf dieser Seite können Trigger anhand von Eingabedaten ganz realistisch getestet werden. Diese Eingabedaten kriegt man etwa aus dem Quelltext der Seite (Copy & Paste) bzw. aus Adressleiste, etc.

Zusätzlich können Debugausgaben angeschaltet werden. Auf diese Weise kann man zuschauen, wie der Trigger-Compiler/Interpreter arbeitet und damit Fehler in seiner Trigger-Logik nachvollziehen. Vorsicht: Debug-Ausgaben auf *allen* Triggern erzeugen sehr viele Ausgaben.

Siehe auch Erklärungen der Debugausgaben

Eingabedaten

URL (Adresse der Seite):

Content (Quelltext der Seite):

Zu testende Trigger

Trigger-IDs: In diesem Eingabefeld können beliebig viele Trigger-IDs genannt werden (pro Zeile eine ID), die dann in dieser Reihenfolge getestet werden. Wenn leer, werden alle Trigger ausprobiert. Gebe unsinnigen Inhalt (etwa ---) ein, um gar keinen Trigger zu testen (etwa, um nur deinen eigenen Trigger zu testen, siehe unten)

Zusätzlich kann in diesem Eingabefeld ein eigener Trigger live ausprobiert werden. Damit können Formulierungen ganz ohne Speicherung ausprobiert werden. Dieser Trigger kriegt eine zufällige temporäre ID und wird als letzter Trigger (nach obiger Liste) getestet.
